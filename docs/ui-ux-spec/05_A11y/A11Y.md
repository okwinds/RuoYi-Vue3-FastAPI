# 可访问性（A11y）

本仓库前端整体依赖 Element Plus 的可访问性基础，但仍存在一些“全局样式层面的焦点可见性”与“图标语义”等缺口。本文档描述**现状**与**复刻要求**（保持一致），并给出建议改进项（供后续迭代）。

---

## 1) 现状（Current State）

- 图标组件 `SvgIcon` 默认 `aria-hidden="true"`：图标不被读屏读取（适合装饰性图标）。
- 全局样式中存在 `div:focus { outline: none; }`：可能降低键盘用户的焦点可见性。
- 表单/按钮/输入控件大多使用 Element Plus，默认键盘交互与 aria 支持由组件库提供。

---

## 2) 复刻要求（Replication Requirements）

若复刻实现替换 UI 框架/组件库，至少要保证：

- 键盘可达：主要交互（菜单、搜索表单、表格操作按钮、分页、弹窗）可通过键盘操作完成
- 焦点管理：弹窗打开后焦点进入弹窗；关闭后回到触发点（Element Plus 默认行为需保持）
- 对比度：暗黑模式下文字与背景对比度不明显下降（本项目通过 CSS 变量覆盖实现）
- 图标语义：装饰性图标应隐藏；承载语义的图标必须有可读文本（例如按钮的 `aria-label` 或可见文本）

---

## 3) 建议改进项（非本次规格交付的强制项）

- 为全局 `focus-visible` 补充样式（避免 `outline: none` 带来的可访问性下降）
- 主题切换按钮（sunny/moon 图标）增加可读文本或 `aria-label`
- 错误页与空状态图片补充更明确的 alt 文本（当前 404 图片 alt=404，语义较弱）
